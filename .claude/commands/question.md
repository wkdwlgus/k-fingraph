# /question — 본 흐름을 끊지 않는 사이드 질문

지금부터 들어오는 질문은 진행 중이던 작업과 별개의 호기심·용어 확인·
배경 설명 요청이다. 다음 규칙을 따른다:

1. **질문에만 답하라.** 진행 중이던 작업의 다음 스텝을 실행하거나
   파일을 수정하지 말 것. 코드 변경 제안도 하지 말 것 (질문이 명시적으로
   요구하지 않는 한).
2. **답변은 짧게.** 2~5문장. 더 깊이 원하면 사용자가 follow-up.
3. **답변 끝에 "↩ 돌아갈 지점" 한 줄을 붙여라.** 직전에 무엇을 하고
   있었고, 사용자의 다음 액션이 무엇이었는지 한 줄로 상기시켜라.
   예: `↩ 돌아갈 지점: Day 2 첫 단위 plan 승인 대기 중 (go/ok 입력 시 구현 시작).`
4. **본 흐름의 결정·계획·코드는 손대지 말 것.** 질문 답변은 별도
   레이어. `tasks/current.md`, plan, 진행 중인 변경 등은 그대로 둔다.
5. 질문이 사실은 본 작업의 방향을 바꾸는 결정이라면 (예: "이 접근
   다시 생각해보자"), 답변 마지막에
   "이건 사이드 질문이 아니라 본 계획에 영향을 줍니다. /plan 다시 돌릴까요?"
   라고 표시하라.
