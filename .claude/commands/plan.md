# /plan — 새 작업 시작 전 계획 수립

다음 순서로 진행하라:

1. `tasks/current.md`를 읽고 다음에 할 작업 확인
2. 해당 작업이 손대야 할 파일/모듈 목록을 사용자에게 제시
3. 의존성: 어떤 함수/스키마/문서를 먼저 손봐야 하는가?
4. 변경할 함수의 시그니처를 미리 적어 사용자에게 보여라
5. 테스트 전략: unit/integration/e2e 중 어디에 무엇을 추가할 것인가
6. 예상 작업 시간을 추정하라 (30분/1시간/2시간/반나절)
7. **사용자가 "go" 또는 "ok" 라고 명시적으로 승인하기 전에는 코드를 수정하지 말 것**

작성 결과물:
- 변경 대상 파일 리스트
- 새 함수 시그니처
- 테스트 추가 계획
- 예상 시간

승인 후에야 실제 구현 시작.
